# Sprout Agent Instructions

Before changing Sprout code or specs, use the repo-local skill:

`skills/sprout-spec-guardian/SKILL.md`

That skill is the short constitution for this repo. It tells you which `spec/` docs to read, how to restate scope before coding, what failure states are required, and what product drift is forbidden.

Minimum rule: do not build from generic finance-app instincts. Read the relevant specs, restate what you are building and not building, surface unknowns instead of inventing values, and preserve `spec/user_stories_regression_invariants.md`.
